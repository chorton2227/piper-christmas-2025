Example 4: Navigation

‚ùå Non-Semantic

        <li><a href="#contact">Contact</a></li>

Not terrible, but not semantic either.

‚úÖ Semantic

    <ul>

Using <nav> tells screen readers: "This is navigation!" and they can skip to it instantly.

Quick Decision Guide

When to Use Semantic Elements

Use semantic elements when the content has meaning:

  • ‚úÖ Page header? Use <header>
  • ‚úÖ Navigation menu? Use <nav>
  • ‚úÖ Main content? Use <main>
  • ‚úÖ Blog post? Use <article>
  • ‚úÖ Thematic section? Use <section>
  • ‚úÖ Sidebar? Use <aside>
  • ‚úÖ Footer? Use <footer>
  • ‚úÖ Important text? Use <strong>
  • ‚úÖ Emphasized text? Use <em>

When Divs Are Fine

Use divs for pure layout/styling with no semantic meaning:

  • ‚úÖ Container for centering content
  • ‚úÖ Wrapper for applying background color
  • ‚úÖ Grid or flexbox layout containers
  • ‚úÖ Styling hooks with no semantic purpose
<!-- Divs for layout (fine!) -->
<div class="container">
    <div class="card">
        <!-- Semantic elements for content -->
        <article>
            <h3>Card Title</h3>
            <p>Card content...</p>
        </article>
    </div>
</div>

Real-World Impact

üîç Search Engine Optimization (SEO)

Search engines like Google prefer semantic HTML because they can better understand your content structure.

  • Articles in <article> tags are recognized as content
  • Navigation in <nav> is identified as site structure
  • <time> elements help with date-based searches
  • Better SEO = more visitors to your site!

‚ôø Accessibility

Screen readers use semantic elements to help users navigate:

  • Users can jump directly to <main> content
  • They can list all <nav> elements
  • They can skip to specific <section> or <article>
  • <strong> and <em> are announced with emphasis

Making your site accessible isn't optional‚Äîit's essential!
