Understanding Z-Index

Higher z-index values appear in front of lower values

z-index: 1
z-index: 2
z-index: 3
.layer-1 { position: relative; z-index: 1; } .layer-2 { position: relative; z-index: 2; } .layer-3 { position: relative; z-index: 3; } /* Higher number = appears in front */

üéØ Stacking Context

Each positioned element creates its own stacking context

Parent z-index: 1
Child z-index: 100
Parent z-index: 2
Child z-index: 1
/* Parent 2 (z-index: 2) appears in front of Parent 1 (z-index: 1) Even though Child in Parent 1 has z-index: 100! Children's z-index only matters within their parent's context */

üí° A child's z-index is relative to its parent's stacking context

Real World: Modal Dialog

Modals use high z-index to appear above everything

Modal Dialog

This modal has a high z-index to appear above all content.

The backdrop has z-index: 1000, the modal has z-index: 1001.

.modal-backdrop { position: fixed; top: 0; left: 0; right: 0; bottom: 0; background: rgba(0, 0, 0, 0.5); z-index: 1000; /* High z-index */ } .modal-content { position: relative; z-index: 1001; /* Even higher */ }

üìã Z-Index Best Practices

‚úÖ
Use consistent values

Use multiples of 10 or 100: 10, 20, 30 or 100, 200, 300

‚úÖ
Create a z-index scale

Define standard layers: content (1), dropdowns (100), modals (1000)

‚ùå
Avoid z-index wars

Don't keep increasing values randomly (9999, 99999)

‚úÖ
Position is required

Remember: z-index only works on positioned elements!

üìä Common Z-Index Scale

-1 Background elements
0-10 Normal content
100-200 Dropdowns, tooltips
500-600 Sticky headers
1000+ Modals, overlays
