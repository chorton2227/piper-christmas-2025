# Exercise 2: Plan Your Repository

## Objective

Plan how you'll organize and describe your poker game repository on GitHub.

## Instructions

### Step 1: Choose Your Repository Name

Your repository name will be part of your URL: `https://yourusername.github.io/[repo-name]`

**Rules for repository names:**
- All lowercase
- No spaces (use hyphens `-` instead)
- Letters, numbers, and hyphens only
- Should describe the project

**Good examples:**
- `poker-game`
- `texas-holdem`
- `holdem-poker`
- `poker-showdown`

**Bad examples:**
- `My Poker Game` (has spaces and capitals)
- `game123` (not descriptive)
- `final_project` (generic, underscores less common)
- `PokerGame` (capitals not conventional)

### Step 2: Write Your Repository Description

GitHub lets you add a short description (under 100 characters) that appears at the top of your repository.

**Good description formula:**
`[What it is] + [technology used] + [special feature]`

**Examples:**
- "A fully-featured Texas Hold'em poker game built with HTML, CSS, and JavaScript"
- "Texas Hold'em poker game with AI opponents and realistic betting"
- "Browser-based poker game featuring Texas Hold'em rules and animations"
