🐛 Common Errors & How to Fix Them

📢 Important: Open your browser's Developer Tools (Press F12) to see errors in the Console!

Error 1: ReferenceError - Variable Not Defined

❌ Error: ReferenceError: myVariable is not defined

What it means: You're trying to use a variable that doesn't exist.

❌ Wrong (causes error):
console.log(myVariable); // Variable doesn't exist!
✅ Fixed:
let myVariable = "Hello"; console.log(myVariable); // Now it exists!
💡 Common Causes:
  • Typo in variable name
  • Forgot to declare the variable
  • Variable is out of scope
  • Using variable before it's declared

Error 2: TypeError - Not a Function

❌ Error: TypeError: something is not a function

What it means: You're trying to call something that isn't a function.

❌ Wrong:
let number = 5; number(); // number is not a function!
✅ Fixed:
let number = 5; console.log(number); // Just use it, don't call it // OR if you meant to have a function: let calculate = function() { return 5; }; calculate(); // Now it's a function!

Error 3: TypeError - Cannot Read Property

❌ Error: Cannot read property 'x' of undefined/null

What it means: Trying to access a property of something that doesn't exist.

❌ Wrong:
let person; console.log(person.name); // person is undefined!
✅ Fixed:
let person = { name: "Alice" }; console.log(person.name); // person exists now! // OR check if it exists first: if (person) { console.log(person.name); } // OR use optional chaining (modern way): console.log(person?.name);

Error 4: SyntaxError - Unexpected Token

❌ Error: SyntaxError: Unexpected token

What it means: JavaScript doesn't understand your code - grammar mistake!

❌ Wrong (multiple syntax errors):
// Missing comma let person = { name: "Alice" age: 25 // Missing comma here! }; // Missing closing bracket let numbers = [1, 2, 3; // Missing closing quote let message = "Hello; // Typo in keyword functoin sayHi() { // Should be "function" console.log("Hi"); }
✅ Fixed:
// Added comma let person = { name: "Alice", age: 25 // Comma added! }; // Fixed bracket let numbers = [1, 2, 3]; // Fixed quote let message = "Hello"; // Fixed keyword function sayHi() { console.log("Hi"); }
💡 How to Prevent:
  • Use an editor with syntax highlighting
  • Match every opening bracket/quote with a closing one
  • Add commas between object properties and array items
  • Check spelling of keywords (function, if, for, etc.)

Error 5: Logic Errors (No Error Message!)

❌ The Hardest Type: Wrong Results, No Error

What it means: Code runs but gives wrong answer.

❌ Wrong Logic #1: Using = instead of ===
let age = 25; if (age = 18) { // ASSIGNS 18, doesn't compare! console.log("You are 18"); // Always runs! } console.log(age); // age is now 18! Bug!
✅ Fixed:
let age = 25; if (age === 18) { // COMPARES values console.log("You are 18"); // Only runs if age is 18 }
❌ Wrong Logic #2: Forgetting to return
function add(a, b) { a + b; // Forgot to return! } let sum = add(5, 3); console.log(sum); // undefined!
✅ Fixed:
function add(a, b) { return a + b; // Return the result! } let sum = add(5, 3); console.log(sum); // 8
❌ Wrong Logic #3: Off-by-one error
let fruits = ["apple", "banana", "cherry"]; for (let i = 1; i < fruits.length; i++) { console.log(fruits[i]); // Starts at 1, skips "apple"! }
✅ Fixed:
let fruits = ["apple", "banana", "cherry"]; for (let i = 0; i < fruits.length; i++) { console.log(fruits[i]); // Starts at 0, gets all items! }

Error 6: Infinite Loop (Freezes Browser!)

⚠️ Warning: Can Crash Your Browser!

What it means: Loop never ends because condition is always true.

❌ Wrong (DON'T RUN THIS!):
let i = 0; while (i < 10) { console.log(i); // Forgot to increment i! Loop never ends! }
✅ Fixed:
let i = 0; while (i < 10) { console.log(i); i++; // Increment so loop can end! }
💡 How to Fix Frozen Browser:
  • Close the browser tab
  • Press ESC key immediately
  • Use Task Manager to close browser
  • Always check your loop increments!

Error 7: Array Index Errors

❌ Problem: Accessing index that doesn't exist
let arr = [1, 2, 3]; console.log(arr[5]); // undefined (not an error, but unexpected) console.log(arr[-1]); // undefined (negative indexes don't work)
✅ Fixed:
let arr = [1, 2, 3]; // Check if index is valid if (index >= 0 && index < arr.length) { console.log(arr[index]); } else { console.log("Index out of bounds!"); } // Or use optional chaining console.log(arr[5] ?? "Not found");

🔍 General Debugging Tips

When you see an error:
  1. Read the error message - it tells you what and where!
  2. Check the line number - go to that line in your code
  3. Look for typos - most common cause
  4. Check brackets/quotes - are they all closed?
  5. Add console.log - see what values are
  6. Simplify the code - break it into smaller pieces
  7. Take a break - fresh eyes help!
Remember: Every programmer makes these mistakes! The difference is knowing how to fix them. Practice makes perfect! 💪
