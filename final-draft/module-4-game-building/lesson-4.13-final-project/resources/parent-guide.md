## Technical Concepts Simplified

### For Your Understanding

**Game State:** Like a save file in a video game - stores all current game information

**Hand Evaluation:** Determining who has the better poker hand (like comparing scores)

**AI Decision Making:** Computer players using if/else logic to make choices

**Async/Await:** Waiting for animations to finish before continuing (like taking turns)

**DOM Manipulation:** Updating what you see on screen (like moving pieces on a board)

## Warning Signs & Solutions

| Warning Sign | What It Means | Solution |
|--------------|---------------|----------|
| Copying code blindly | Not understanding concepts | Have them explain each line |
| Analysis paralysis | Overwhelmed by complexity | Start with smallest piece |
| Spaghetti code | Lack of organization | Help create structure |
| No testing | Assuming code works | Run and test frequently |
| Burnout | Too much too fast | Enforce breaks |

## Frequently Asked Questions

**Q: How much should I help with the code?**
A: Guide with questions, don't write code for them. Help them think through problems.

**Q: What if they want to give up?**
A: Encourage a break, then help identify one small achievable goal.

**Q: Is it okay to use online resources?**
A: Yes! Using documentation and examples is part of programming. Just ensure they understand what they're using.

**Q: Should they start over if code gets messy?**
A: Usually no - refactoring (cleaning up) is a valuable skill. Complete it first, then improve.

**Q: How long should debugging take?**
A: 15-30 minutes solo, then encourage asking for help if stuck.
