### 1. Planning Phase
**Help them choose a topic:**
- What are they passionate about?
- Gaming, sports, hobbies, pets, books, movies?
- Something they can write 3-5 paragraphs about
- Topic should have visual potential (colors, images)

**Ask guiding questions:**
- "What would you want to see on a website about this topic?"
- "What colors represent your topic?"
- "What sections would be most important?"

### 2. During Development

**Encourage breaks:**
- CSS can be frustrating when things don't work
- Suggest 10-minute breaks every hour
- Fresh eyes often spot issues quickly

**Help debug (without doing it for them):**
- Ask: "What were you trying to achieve?"
- Ask: "What's actually happening?"
- Ask: "What have you tried so far?"
- Guide them to check browser developer tools

**Celebrate small wins:**
- "That gradient looks great!"
- "Nice hover effect on those buttons!"
- "The spacing looks much better now!"

### 3. Common Challenges & Solutions

**"My fixed header is covering my content!"**
- Solution: Add margin-top to first section equal to header height

**"My flexbox items aren't the same size!"**
- Solution: Add `flex: 1` to flex items

**"Nothing is showing up!"**
- Check: Is CSS file linked correctly?
- Check: Are class names spelled correctly?
- Check: Are there any typos in CSS?

**"My colors look bad together!"**
- Suggest: Use a color palette generator
- Suggest: Stick to 3-5 colors maximum
- Suggest: Use lighter backgrounds with darker text

**"It looks good on my computer but bad on mobile!"**
- Check: Is viewport meta tag in HTML head?
- Check: Are media queries included?
- Suggest: Test with browser dev tools mobile view

### 4. Testing Together

**Test responsiveness:**
- Resize browser window together
- Check on actual mobile device if available
- Verify navigation works at all sizes

**Test interactions:**
- Do all buttons have hover effects?
- Do links work correctly?
- Is text readable on all backgrounds?

**Test across browsers:**
- Try Chrome, Firefox, Edge
- Usually consistent, but good practice

## üé® Design Feedback Tips

### Positive Reinforcement:
- "I love how the colors match your theme!"
- "The card hover effects are really smooth!"
- "The layout looks very professional!"
- "Great job making it responsive!"

### Constructive Feedback:
Instead of: "That looks bad"
Try: "What if we tried a different color there?"

Instead of: "The spacing is wrong"
Try: "Could we add a bit more space between those sections?"

Instead of: "That's too small"
Try: "Would it be easier to read if it was a bit larger?"
