Getting Started with HTML

Welcome to HTML Documentation! Whether you're brand new to web development or looking to master semantic HTML, you're in the right place.

HTML is the foundation of the web. It provides the structure and meaning to web content, making it accessible to everyone—from search engines to screen readers to developers maintaining your code years from now.

HTML Elements Reference

<header> Element

Category: Sectioning Content | HTML5

Description

The <header> element represents introductory content for its nearest ancestor sectioning content or sectioning root element. It typically contains heading elements, logos, navigation, or search forms.

Code Example

<header>
    <h1>My Website</h1>
    <nav>
        <a href="#home">Home</a>
        <a href="#about">About</a>
    </nav>
</header>

When to Use

  • Page headers with site title and main navigation
  • Article headers with title, author, and date
  • Section headers with introductory content

Important: A page can have multiple <header> elements—one for the page and others within articles or sections.

Supported in all modern browsers | Last updated: Jan 15, 2025

<nav> Element

Category: Sectioning Content | HTML5

Description

The <nav> element represents a section containing navigation links, either to other pages or to parts within the current page. It helps screen readers and search engines identify the main navigation areas.

Code Example

<nav>
    <ul>
        <li><a href="#home">Home</a></li>
        <li><a href="#about">About</a></li>
        <li><a href="#contact">Contact</a></li>
    </ul>
</nav>

When to Use

  • Main site navigation (primary menu)
  • Table of contents for long articles
  • Breadcrumb navigation
  • Footer navigation (secondary links)

Note: Not all links need to be in <nav>—only major navigation blocks.

Supported in all modern browsers | Last updated: Jan 15, 2025

<article> Element

Category: Sectioning Content | HTML5

Description

The <article> element represents a self-contained composition that could be independently distributed or reused. Think blog posts, news articles, forum posts, or product cards.

Code Example

<article>
    <header>
        <h2>Article Title</h2>
        <time>Jan 15, 2025</time>
    </header>
    <p>Article content...</p>
    <footer>
        <p>By Author Name</p>
    </footer>
</article>

When to Use

  • Blog posts and news articles
  • Forum posts and comments
  • Product cards in a catalog
  • User-submitted content

Test: Could this content be distributed via RSS? If yes, use <article>!

Supported in all modern browsers | Last updated: Jan 15, 2025

<section> Element

Category: Sectioning Content | HTML5

Description

The <section> element represents a thematic grouping of content, typically with a heading. Use it to divide content into logical parts.

Code Example

<section>
    <h2>Our Services</h2>
    <p>We offer the following...</p>
    <ul>
        <li>Service 1</li>
        <li>Service 2</li>
    </ul>
</section>

When to Use

  • Thematic content grouping (chapters, tabs)
  • Major page sections (services, features)
  • Within articles to divide content

Tip: If it needs a heading, <section> is usually the right choice!

Supported in all modern browsers | Last updated: Jan 15, 2025

<strong> Element

Category: Text-level Semantics | HTML

Description

The <strong> element indicates that text has strong importance, seriousness, or urgency. Browsers typically display it in bold, but its meaning is semantic, not just visual.

Code Example

<p>
    <strong>Warning:</strong> 
    This action cannot be undone.
</p>

When to Use

  • Important information or warnings
  • Key points in instructional content
  • Urgent messages

Difference: Use <strong> for importance, <b> for visual boldness only.

Supported in all browsers | Last updated: Jan 15, 2025

<time> Element

Category: Text-level Semantics | HTML5

Description

The <time> element represents a specific date, time, or duration in a machine-readable format. This helps search engines and screen readers understand temporal information.

Code Example

<p>
    Published: 
    <time datetime="2025-01-15">
        January 15, 2025
    </time>
</p>

When to Use

  • Publication dates for articles
  • Event dates and times
  • Last updated timestamps
  • Durations (using duration format)

Always include the datetime attribute! Format: YYYY-MM-DD or YYYY-MM-DDTHH:MM:SS

Supported in all modern browsers | Last updated: Jan 15, 2025

Best Practices

Accessibility Tips

Accessible HTML is better HTML. When you use semantic elements properly, you're automatically making your site more accessible to people using screen readers and other assistive technologies.

DO:

  • Use semantic elements — <nav>, <article>, <main> help screen readers navigate
  • Always add labels to form inputs — Use <label> elements with for attributes
  • Provide alt text for images — The alt attribute describes images to non-visual users
  • Use proper heading hierarchy — <h1> → <h2> → <h3> (don't skip levels!)
  • Test with keyboard navigation — Press Tab to move through links and forms

DON'T:

  • Use divs for everything (they provide no semantic meaning)
  • Skip heading levels (h1 → h3 is bad!)
  • Rely on color alone to convey information
  • Create keyboard traps where users can't tab out

"The power of the Web is in its universality. Access by everyone regardless of disability is an essential aspect."

— Tim Berners-Lee, inventor of the World Wide Web

Remember: About 15% of the world's population lives with some form of disability. Accessible design benefits everyone!

SEO Guidelines

SEO isn't just about keywords—it's about helping search engines understand your content. Semantic HTML is your secret weapon for better search rankings.

Best Practices:

  • Use one <h1> per page — It tells search engines what your page is about
  • Put important content in <main> — Search engines know this is your primary content
  • Use <article> for indexable content — Helps search engines identify distinct pieces of content
  • Use <time> with datetime attributes — Search engines understand when content was published
  • Structure content with proper headings — Creates an outline search engines can understand
  • Use <nav> for navigation — Helps search engines understand your site structure

Key insight: Google's algorithm rewards well-structured, semantic HTML. It's not magic—it's just good code!

Code Organization

Clean, well-organized HTML is easier to maintain, debug, and collaborate on. Future you (and your teammates) will thank you for writing clear, semantic markup.

Guidelines:

  • Use consistent indentation — Two spaces or four spaces, pick one and stick with it
  • Close tags properly — Even though HTML5 is forgiving, proper nesting prevents bugs
  • Add comments for complex sections — Use <!-- comments --> to explain non-obvious code
  • Group related content — Use semantic elements to create logical sections
  • Keep it simple — Don't nest too deeply; aim for flat, readable structures

Pro tip: Use Shift + Alt + F in VS Code to auto-format your HTML!

Common Mistakes to Avoid

1. Using Divs for Everything

Bad: 

<div class="header">
    <div class="nav">...</div>
</div>

Good: 

<header>
    <nav>...</nav>
</header>

2. Multiple <main> Elements

Only one <main> per page! It identifies the primary content.

3. Skipping Heading Levels

Bad: <h1> → <h3> (skipped h2!)

Good: <h1> → <h2> → <h3>

4. Using <b> and <i> Instead of <strong> and <em>

Use semantic elements (<strong>, <em>) instead of purely visual ones (<b>, <i>) when the text has meaning, not just styling.

5. Forgetting datetime Attributes on <time>

Always include datetime so machines can understand dates!

Complete Examples

Example 1: Simple Blog Page

This example demonstrates basic semantic structure for a blog with header, main content, and footer.

Code

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>My Blog</title>
</head>
<body>
    <header>
        <h1>My Awesome Blog</h1>
        <nav>
            <a href="#home">Home</a>
            <a href="#about">About</a>
        </nav>
    </header>
    
    <main>
        <article>
            <header>
                <h2>My First Post</h2>
                <time datetime="2025-01-15">
                    Jan 15, 2025
                </time>
            </header>
            <p>Content goes here...</p>
        </article>
    </main>
    
    <footer>
        <p>© 2025 My Blog</p>
    </footer>
</body>
</html>

Key Takeaways

  • One <main> element wraps the primary content
  • <header> and <footer> at both page and article levels
  • <nav> clearly marks navigation
  • <time> with datetime attribute for dates

Example 2: Documentation Page with Sidebar

This example shows how to create a documentation layout with main content and a sidebar using <aside>.

Code

<body>
    <header>
        <h1>Documentation</h1>
        <nav>...</nav>
    </header>
    
    <main>
        <section id="intro">
            <h2>Introduction</h2>
            <p>...</p>
        </section>
        
        <section id="guide">
            <h2>User Guide</h2>
            <article>
                <h3>Getting Started</h3>
                <p>...</p>
            </article>
        </section>
    </main>
    
    <aside>
        <h2>Table of Contents</h2>
        <nav>
            <ul>
                <li><a href="#intro">Introduction</a></li>
                <li><a href="#guide">User Guide</a></li>
            </ul>
        </nav>
    </aside>
    
    <footer>...</footer>
</body>

Key Takeaways

  • <aside> for supplementary content (table of contents, related links)
  • Multiple <section> elements for different topics
  • Nested <nav> within <aside> is perfectly valid
  • Clear content hierarchy with proper headings
